## Расчёты

**Натальная карта**  
`astrology.ts` рассчитывает планеты через `astronomy-engine`, сидерическую долготу через Lahiri-приближение, накшатру, паду, Раху и Кету. Асцендент считается через местное звёздное время, долготу, широту, наклон эклиптики и пересечение эклиптики с горизонтом.

**Дома**  
Используется Whole Sign: знак Лагны становится 1-м домом, следующие знаки дают дома 2-12. Управители: Mars для Aries/Scorpio, Venus для Taurus/Libra, Mercury для Gemini/Virgo, Moon для Cancer, Sun для Leo, Jupiter для Sagittarius/Pisces, Saturn для Capricorn/Aquarius.

**Вимшоттари Даша**  
Последовательность: Ketu 7, Venus 20, Sun 6, Moon 10, Mars 7, Rahu 18, Jupiter 16, Saturn 19, Mercury 17 лет. Стартовая Махадаша берётся по накшатре Луны, стартовый баланс учитывает положение Луны внутри накшатры. Антардаша считается внутри текущей Махадаши.

**Панчанг**  
`panchang.ts` рассчитывает титхи, накшатру, йогу, карану и вару. Данные используются в дневном отчёте, календаре, сфере дня и блоке “Сейчас”.

**Планетарные Часы**  
`planetaryHours.ts` ищет фактический восход, закат и следующий восход. День и ночь делятся на 12 неравных часов. Для экстремальных широт есть fallback на 06:00/18:00, который помечается в UI.

**Аспекты**  
`aspectCalculations.ts` разделяет натальные и транзитные аспекты. Поддержаны соединение 0, секстиль 60, квадрат 90, трин 120, оппозиция 180. Сила хранится как 0-100. `aspectInterpretations.ts` содержит детальные и fallback-интерпретации.

**Варги**  
`divisionalCharts.ts` считает дробные карты. Специализированные правила есть для D2, D3, D7, D9, D10, D12, D20 и D24; остальные помечаются как generic и не используются как сильный фактор рекомендаций. Для каждой планеты в `PlanetPosition.vargas` сохраняются быстрые показатели D12/D20/D24, которые используются как мягкие дополнительные подсказки в персональных рекомендациях.

**Энергия Дня**  
`energyUtils.ts` даёт детерминированный уровень 0-100. Учитываются дата, Луна, титхи и персональные факторы: Лагна, натальная Луна, текущая Луна, Даша, аспектная сила, погрешность времени.

## Персонализация

Единая точка рекомендаций: `personalRecommendations.ts`.

Вход:

- профиль пользователя;
- дата;
- натальная карта;
- текущая карта/транзиты;
- аспекты;
- панчанг;
- планетарный час.

Выход:

- главная рекомендация;
- чего избегать;
- лучшие сферы дня;
- микро-упайя;
- объяснение причин;
- уровень точности;
- энергия;
- активная Даша и Антардаша.

Принцип: рекомендация должна объяснять причину, а не просто брать текст из шаблона “high/low”.

## Сферы Жизни

`dailyAreasAnalysis.ts` анализирует 8 сфер:

- карьера: дома 6, 10, 11;
- отношения: 5, 7, 12;
- здоровье: 1, 6, 8;
- финансы: 2, 6, 10, 11;
- обучение: 4, 5, 9;
- творчество: 3, 5, 9;
- духовность: 8, 9, 12;
- семья: 2, 4, 7.

Весовая идея: Даша задаёт фон периода, транзиты дают ежедневные триггеры, панчанг задаёт атмосферу дня. Тексты рекомендаций должны ссылаться на эти причины: активная планета, дом, аспект, йога, титхи или точность данных.

## Календарь

`ForecastCalendar.tsx` показывает месяц с понедельника. `forecastCalculations.ts` строит прогноз на локальный полдень выбранного дня. `indexedDB.ts` хранит кэш прогнозов.

`DayForecast` содержит дату, ключ профиля, версию расчёта, уровень энергии, Луну, титхи, йогу, планетарный час, главную рекомендацию, избегание, причины, лучшие сферы, микро-упайю, заметки о точности, предупреждения, благоприятные часы, цвет, иконку и timestamp.

Кэш зависит от профиля, координат, timezone, текущего местоположения, погрешности времени и версии расчёта. Старые прогнозы пересчитываются при изменении ключа.

## Геокодирование И Timezone

`geocoding.ts` использует OpenStreetMap Nominatim:

```text
https://nominatim.openstreetmap.org/search?q=...&format=json&limit=5&addressdetails=1
```

Отправляется только строка поиска места. API ключ не нужен. Результаты не кэшируются, потому что поиск редкий и данные OSM могут обновляться.

`timezones.ts` использует локальную таблицу крупных регионов. Если координаты попадают в регион, точность `matched-region`; если нет, timezone оценивается по долготе и точность `estimated-longitude`. Исторические offset/DST применяются через `Intl` для найденной IANA-зоны.

Ограничение: в проекте нет полноценной polygon-базы timezone. Для границ зон, спорных территорий, океанов и старых дат нужна ручная проверка.

## Хранение И Приватность

`storage.ts` хранит профиль локально. В профиле есть `timeUncertainty` и `timezoneAccuracy`. Текущее местоположение опционально и имеет собственную `timezoneAccuracy`.

Пользователь может:

- экспортировать отчёт в PDF;
- скачать профиль в JSON;
- удалить профиль.

Импорт профиля обратно пока не реализован.

## UI И Компоненты

Общие компоненты:

- `GlassCard.tsx` - единая glass-карточка;
- `ErrorBoundary.tsx` - защита от падений рендера;
- `ErrorDisplay.tsx` - единый вывод ошибок;
- `LoadingSpinner.tsx` - состояния загрузки;
- `PlanetBadge.tsx` - планета с иконкой, цветом и названием.

Хуки:

- `useAstroCalculations.ts` - центральный расчёт натальной карты, транзитов, аспектов, панчанга, планетарного часа и окон времени;
- `useGeocoding.ts` - поиск места, состояния загрузки и ошибки.

Крупные экраны дробятся на под-компоненты. `DailyReport.tsx` уже разбит на `DayHeadline`, `TopRecommendations`, `AspectsToday`, `FavorableWindows`, `LocationContext`.

Дизайн: спокойная светлая тема с фиолетовыми/розовыми/голубыми/зелёными акцентами, glassmorphism, адаптивная сетка, подсказки для терминов, понятные состояния загрузки/ошибки. Не плодить inline-дубли карточек; использовать общие компоненты, когда они подходят.

## Тестовые Сценарии

Критично проверять:

- первый запуск и все 4 шага онбординга;
- поиск места, множественные результаты, пустой запрос, несуществующее место;
- координаты в диапазоне широта -90..90, долгота -180..180;
- дату рождения в прошлом, високосный год, 00:00 и 23:59;
- натальную карту, Асцендент, дома, панчанг, аспекты, Дашу;
- календарь с кэшем, пересчёт при смене профиля/локации;
- работу без точного времени и отображение пониженной точности;
- блок “Сейчас” и fallback планетарных часов;
- ErrorBoundary и retry при ошибках;
- экспорт PDF/JSON и удаление профиля.

Известные ручные проверки:

- Москва: примерно 55.7558, 37.6173, `Europe/Moscow`;
- Санкт-Петербург: примерно 59.9343, 30.3351;
- Владивосток: примерно 43.1332, 131.9113, `Asia/Vladivostok`;
- Нью-Йорк: примерно 40.7128, -74.0060;
- Лондон: примерно 51.5074, -0.1278.

## Известные Ограничения

- Timezone определяется облегчённо, не polygon-базой.
- Lahiri ayanamsha реализована приближённо от J2000.
- PDF с кириллицей может зависеть от поддержки шрифта в браузере/jsPDF.
- Nominatim не исправляет опечатки.
- Нет импорта JSON обратно.
- Нет PWA/offline service worker.
- Нет карт близких, синастрии, голосового интерфейса, “спроси карту”, push-уведомлений.
- Варги generic не должны быть основанием для сильных прогнозов.
